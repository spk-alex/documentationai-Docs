This catalog lists every event published by SmartphoneKey. For each event you'll find the source(s), when it fires, and the key fields in its payload.

For full field-level schema documentation including types and constraints, see the **Event Schema Reference** section in the Events tab.

## Key Management

Events related to digital keys being programmed, removed, or assigned to users.

### KeyAdded

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `KeyAdded` |

Fires when a new digital key credential is programmed into a lock's key slot.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | number | Internal numeric ID of the lock |
| `uuid` | string | UUID of the lock aggregate |
| `keyUuid` | string | Unique identifier of the key credential |
| `keyIndexNumber` | number | Slot index the key was programmed into |
| `timestamp` | date-time | When the key was added |

### TempKeyCreated

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `TempKeyCreated` |

Fires when a temporary access key is created. Temp keys can be time-bounded, limited to a maximum number of uses, and restricted to specific users.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | number | Internal numeric ID of the lock |
| `lockUuid` | string | UUID of the lock aggregate |
| `tempKeyUuid` | string | Unique identifier of the temporary key |
| `createdBy` | string | User ID who created the temp key |
| `validFrom` | date-time | Start of the access window |
| `validUntil` | date-time | End of the access window |
| `accessMode` | string | Access mode — e.g., `once`, `always`, `scheduled` |
| `maxUses` | number\|null | Max uses allowed; null means unlimited |
| `allowedUserIds` | string[] | Users allowed to use this key; empty means any user |

---

### TempKeyUpdated

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `TempKeyUpdated` |

Fires when a temporary key's properties are modified. Only the changed fields are present in the `changes` object.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | number | Internal numeric ID of the lock |
| `lockUuid` | string | UUID of the lock aggregate |
| `tempKeyUuid` | string | Unique identifier of the temporary key |
| `updatedBy` | string | User ID who made the update |
| `changes` | object | Partial update — only changed fields included (`validFrom`, `validUntil`, `accessMode`, `maxUses`, `allowedUserIds`) |

---

## Lock Management

Events related to lock devices and access permissions on locks.

### LockCreated

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `LockCreated` |

Fires when a new lock is registered and linked to an AWS IoT thing. The `spk.api` variant includes an additional `serialNumber` field.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | number | Internal numeric ID of the lock |
| `uuid` | string | UUID of the lock aggregate |
| `thingName` | string | AWS IoT thing name assigned to this lock |
| `thingArn` | string | AWS IoT thing ARN |
| `serialNumber` | string | Hardware serial number (spk.api only) |
| `keys` | object[] | Initial key slots provisioned on the lock |
| `timestamp` | date-time | When the lock was created |

---

### AccessibleLockAdded

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `AccessibleLockAdded` |

Fires when a user is granted access to a lock. The `accessLevel` field indicates the type of access granted.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | string | UUID of the lock |
| `lockName` | string | Human-readable name of the lock |
| `userId` | string | ID of the user who was granted access |
| `grantedBy` | string | ID of the user or system that granted access |
| `accessLevel` | string | Level of access granted — e.g., `owner`, `resident`, `guest` |

---

### AccessibleLockRemoved

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `AccessibleLockRemoved` |

Fires when a user's access to a lock is revoked.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `lockId` | string | UUID of the lock |
| `userId` | string | ID of the user whose access was revoked |
| `removedBy` | string | ID of the user or system that revoked access |
| `removedAt` | date-time | When access was revoked |
| `reason` | string | Reason code for the revocation |

---

## Hub Management

Events related to SmartphoneKey hub devices — the bridge hardware that connects locks to the cloud.

### HubClaimed

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `HubClaimed` |

Fires when a hub device is claimed by an organization and assigned to a site.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `hubId` | string | Unique identifier of the hub |
| `orgId` | string | ID of the organization that claimed it |
| `siteId` | string | ID of the site the hub is deployed at |
| `claimedBy` | string | User ID of the person who performed the claim |
| `claimedAt` | int64 | Unix timestamp (ms) when claimed |

---

### HubProvisioned

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `HubProvisioned` |

Fires when a hub completes IoT provisioning and receives its device certificate from AWS IoT.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `hubId` | string | Unique identifier of the hub |
| `serial` | string | Hardware serial number of the hub |
| `thingName` | string | AWS IoT thing name assigned to this hub |
| `model` | string | Hardware model identifier |
| `provisionedAt` | int64 | Unix timestamp (ms) when provisioning completed |

---

### HubRoleSet

| | |
|--|--|
| **Sources** | `customer.events`, `spk.api` |
| **detail-type** | `HubRoleSet` |

Fires when a hub's role is configured within a site. A site can have one primary hub and multiple secondary hubs.

**Key payload fields (`data`):**

| Field | Type | Description |
|-------|------|-------------|
| `hubId` | string | Unique identifier of the hub |
| `isPrimary` | boolean | Whether this hub is the primary for the site |
| `setBy` | string | User ID who set the role |
| `setAt` | int64 | Unix timestamp (ms) when the role was assigned |

---

### HubSuspended

## Authentication Events

### UserSignup

| | |
|--|--|
| **Source** | `auth0.signup` |
| **detail-type** | `UserSignup` |

Fires when a user completes registration via Auth0. Published directly from the Auth0 EventBridge integration.

This is distinct from `UserCreated`: `UserSignup` represents the authentication identity being created, while `UserCreated` represents the application user record.

**Payload fields (top-level `detail`):**

| Field | Type | Description |
|-------|------|-------------|
| `user_id` | string | Auth0 user ID (format: `auth0\|<id>`) |
| `email` | string | Email address used for registration |

## IoT Events

### ShadowUpdate

| | |
|--|--|
| **Source** | `spk.iot.shadow` |
| **detail-type** | `Shadow Update` |

Fires when an IoT hub device updates its AWS IoT Device Shadow. Contains the full shadow state snapshot including desired state, reported state, and the delta.

**Payload fields (top-level `detail`):**

| Field | Type | Description |
|-------|------|-------------|
| `thingName` | string | AWS IoT thing name of the device |
| `partnerId` | string\|null | Partner associated with this device |
| `version` | number | Shadow document version |
| `desiredState` | object\|null | Target state set by the application |
| `reportedState` | string | State the device reported back (JSON string) |
| `deltaState` | string | Difference between desired and reported state (JSON string); non-empty when the device hasn't applied all changes |
| `metadata` | string | AWS IoT shadow metadata with timestamps per field (JSON string) |
