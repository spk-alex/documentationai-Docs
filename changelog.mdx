## Upgrade Instructions

Follow these steps to apply the latest version across your properties.

<Steps>
  <Step title="Check Compatibility" icon="check">
    Verify your hardware firmware supports the new version in the dashboard.
  </Step>
  <Step title="Backup Data" icon="database">
    Export access logs and user mappings.
  </Step>
  <Step title="Deploy Update">
    <Tabs>
      <Tab title="Dashboard" icon="settings">
        Select properties and click **Update Now**.
      </Tab>
      <Tab title="CLI" icon="terminal">
        ```bash
        smartphonekey update --properties=all --version=latest --api-key=YOUR_API_KEY
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Verify" icon="search">
    Test door unlocking with Apple and Google Wallet on multiple devices.
  </Step>
</Steps>

## Version History

<Update label="2024-10-15" description="v2.1.0" tags={["feature", "improvement"]}>

## New Features

- Added dynamic guest pass expiration tied to rental booking systems.
- Introduced bulk user import from CSV for property managers.

## Improvements

- Reduced unlock latency by `>30%` with optimized Bluetooth Low Energy handling.
- Enhanced dashboard analytics for access pattern visualization.

## Bug Fixes

- Fixed intermittent failures in Google Wallet provisioning for Android 14+ devices.

````jsx
// Example: New bulk import API usage
const users = await smartphonekey.importUsers({
  file: 'tenants.csv',
  propertyId: 'prop_123'
});
````

</Update>

<Update label="2024-09-20" description="v2.0.0" tags={["feature", "breaking"]}>

## New Features

- Apple Wallet Express Mode support for hands-free entry.
- Integration with Airbnb and Vrbo for automatic guest key provisioning.

## Breaking Changes

- Updated API authentication to require `api-key-v2` header instead of legacy tokens.

<Callout kind="alert">
  Migrate your integrations: Replace `{legacy_token}` with `api-key-v2: YOUR_NEW_API_KEY`.
</Callout>

## Bug Fixes

- Resolved door sensor desync after prolonged idle periods.

```javascript
// Updated auth header
fetch('/api/doors/unlock', {
  headers: {
    'api-key-v2': 'YOUR_NEW_API_KEY'
  }
});
```

</Update>

<Update label="2024-08-05" description="v1.9.0" tags={["security", "bugfix"]}>

## Security Updates

- Patched potential replay attack vectors in NFC handover.
- Enforced stricter rate limiting on unlock endpoints (`<100ms` response guaranteed).

## Improvements

- Added audit logs for all Wallet key revocations.

## Bug Fixes

- Corrected iOS Wallet key duplication during user transfers.

<CodeGroup tabs="JavaScript,Python">

</Update>
