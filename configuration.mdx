## Managing User Roles and Permissions

Assign roles to control what users can do, such as granting residents door access or allowing managers to add guests.

<Steps>
  <Step title="Access Admin Dashboard" icon="monitor">
    Log in at `https://admin.smartphonekey.com` with your admin credentials.
  </Step>
  <Step title="Navigate to Users" icon="users">
    Click **Users** in the sidebar, then select **Roles**.
  </Step>
  <Step title="Create or Edit Role" icon="edit-3">
    Click **New Role**, enter a name like `Resident`, and check permissions.
  </Step>
  <Step title="Assign to User" icon="check-circle">
    Go to a user's profile and select the role from the dropdown.
  </Step>
</Steps>

Use this table to understand default permissions:

| Role       | View Users | Add Guests | Manage Doors | Generate Reports |
|------------|------------|------------|--------------|------------------|
| Admin      | Yes        | Yes        | Yes          | Yes              |
| Manager    | Yes        | Yes        | Yes          | Yes              |
| Resident   | No         | No         | No           | No               |
| Guest      | No         | No         | No           | No               |

## Setting Up Access Schedules and Zones

Define when and where users can enter. Schedules apply to specific doors or building zones.

<Tabs>
  <Tab title="Web Dashboard" icon="monitor">
    <Steps>
      <Step title="Create Schedule">
        In **Access** > **Schedules**, click **New Schedule**.
      </Step>
      <Step title="Set Time Window">
        Select days and hours, e.g., `{start: "09:00", end: "18:00"}` weekdays.
      </Step>
      <Step title="Assign to Zone">
        Link to doors like `Front Entrance` or zones like `Building A`.
      </Step>
    </Steps>
  </Tab>
  <Tab title="API Configuration" icon="code">
    Use the API to automate schedules.

    <CodeGroup tabs="JavaScript,cURL">
      ```javascript
      const response = await fetch('https://api.smartphonekey.com/v1/schedules', {
        method: 'POST',
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          name: 'Weekday Access',
          zoneId: 'zone_123',
          schedule: {
            days: ['mon', 'tue', 'wed', 'thu', 'fri'],
            start: '09:00',
            end: '18:00'
          }
        })
      });
      ```
      ```bash
      curl -X POST https://api.smartphonekey.com/v1/schedules \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "name": "Weekday Access",
          "zoneId": "zone_123",
          "schedule": {
            "days": ["mon", "tue", "wed", "thu", "fri"],
            "start": "09:00",
            "end": "18:00"
          }
        }'
      ```
    </CodeGroup>
  </Tab>
</Tabs>

## Configuring Notifications and Reports

Set up email or SMS alerts for key events and schedule automated reports.

<Expandable title="Notification Types" default-open="true">
  - **Access Denied**: Triggers on failed unlocks.
  - **Door Left Open**: Alerts after `{timeout: "5m"}`.
  - **Low Battery**: For hardware readers.
</Expandable>

Configure via API:

<ParamField path="scheduleId" param-type="string" required="true">
  Target schedule identifier.
</ParamField>

<ParamField body="channels" param-type="array" required="false">
  Array like `["email", "sms"]`.
</ParamField>

## Advanced Security Settings

Enable features like two-factor authentication or geofencing for high-security properties.

<ExpandableGroup>
  <Expandable title="Geofencing Setup">
    Define a `{radius: "50m"}` around the property. Users outside this area cannot unlock doors.
  </Expandable>
  <Expandable title="Audit Logs">
    View all access attempts in **Reports** > **Logs**. Export as CSV for compliance.
  </Expandable>
</ExpandableGroup>

<Callout kind="alert">
  Test all configurations in a staging environment before production rollout.
</Callout>

These settings ensure secure, efficient operations tailored to your property. Review logs weekly to monitor usage.
